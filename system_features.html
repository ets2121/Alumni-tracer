Detailed Analytics & Computation Logic

Transparency on how system metrics are calculated is essential for accurate accreditation reporting. Below are the specific formulas used by the backend `ReportController`.

Employment Rate

Calculates the percentage of alumni who are currently employed.

(Employed Count / Total Alumni) * 100
  • Rounded to 1 decimal place.
  • Employed Count: Users with status 'Employed'.

Overseas Employment

Percentage of employed alumni working abroad.

(Overseas Count / Employed Count) * 100
  • Overseas Count: Users with work_location = 'Overseas'.
  • Denominator is Employed users only, not all alumni.

Data Integrity Score

Measures the completeness of the alumni dataset.

((Total - Incomplete) / Total) * 100

A profile is marked Incomplete if ANY are missing:

  • Field of Work
  • Work Status (e.g., Permanent)
  • Establishment Type (e.g., Private)
  • Work Location

Alumni Profile Structure

The profile module allows alumni to manage their personal and professional data. Field visibility and editability are strictly controlled.

1. Registration & Core Identity (Read-Only after creation)
Field Name Description Editability
First / Last Name Legal name of the alumni. Restricted (Admin only)
Course / Program Academic program graduated from (e.g., BSIT). Restricted (Admin only)
Batch Year Year of graduation. Restricted (Admin only)
Student ID / Proof Uploaded image for identity verification. One-time upload
2. Professional Data (User Updateable)

This data feeds directly into the "Detailed Labor" and "Integrity Score" reports.

Field Options / Format Analytics Impact
Employment Status Employed, Unemployed, Ongoing Studies Primary Employment Rate
Work Status Permanent, Contractual, Job Order, Self-Employed Stability Matrix
Establishment Type Public (Gov), Private Sector Analysis
Work Location Local, Overseas Brain Drain / Migration Stats
Company Info Name, Position, Address (Free text) Detailed Labor List

Tracer Survey Module

The system utilizes a Dynamic Form Builder for tracer studies, allowing administrators to create complex surveys without coding.

Conditional Logic

Questions can show/hide based on previous answers.

Example: "If Unemployed" → Show "Reason for unemployment". Hide "Salary".

Dynamic Tables

Allows alumni to enter multiple rows of data.

Used for: List of trainings attended, List of awards received.

Matrix Grids

Complex checkbox matrices for skill assessment.

Example: Rate skills (Communication, Technical) vs (Learned in College, Used in Work).

Reporting & Exporting Features

All reports can be filtered by Date, Course, Batch, and Employment Status.

Export Formats
